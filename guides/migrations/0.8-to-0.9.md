# FUSE Migration: 0.8.x -> 0.9.0

This guide covers required updates when moving from FUSE `0.8.x` to `0.9.0`.

## 1) HTML attr shorthand now accepts expression values

In `0.8.x`, HTML attr shorthand accepted string literals only.
In `0.9.0`, attr values are full expressions.

Before (`0.8.x`):

```fuse
link({"rel": "stylesheet", "href": css})
```

After (`0.9.0`):

```fuse
link(rel="stylesheet" href=css)
```

## 2) Commas are rejected between HTML attrs

HTML tag/component attrs must be layout-separated without commas.

Before (`0.8.x`):

```fuse
div(class="card", id="main"):
  "ok"
```

After (`0.9.0`):

```fuse
div(class="card" id="main"):
  "ok"
```

## 3) Map-literal HTML attrs are rejected

Map literals are no longer valid in HTML attr position.
Use named attrs instead.

Before (`0.8.x`):

```fuse
button({"type": "submit", "class": btn_class})
```

After (`0.9.0`):

```fuse
button(type="submit" class=btn_class)
```

## 4) New diagnostic codes in JSON mode

When using `--diagnostics json`, HTML attr migration failures include:

- `FUSE_HTML_ATTR_MAP`: map literal used in HTML attr position
- `FUSE_HTML_ATTR_COMMA`: comma used between HTML attrs

## 5) Scope of break

- This change affects HTML tag/component attr shorthand call positions.
- Runtime HTML rendering behavior is unchanged after successful compilation.
- Non-HTML syntax (for example struct literals and ordinary map literals) remains valid.
