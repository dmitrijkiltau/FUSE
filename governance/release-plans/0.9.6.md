# FUSE 0.9.6 Implementation Plan: HTTP Client API

## Overview
This document outlines the milestones for implementing the `http.*` client API in FUSE, targeting version 0.9.6. The implementation follows the strict AST/Native/JIT parity requirements and the capability-based security model inherent to the FUSE runtime.

## Milestone 1: API Design & Capability Model
- [ ] Define the exact standard library signatures for `http.request`, `http.get`, and `http.post`.
- [ ] Establish error handling semantics (e.g., non-`2xx` status codes returning `Err`, timeout behavior).
- [ ] Restrict initial scope to `http://` only to simplify the initial network stack, failing fast on `https://`.
- [ ] Define the `requires network` capability requirement in the specification.

## Milestone 2: Compiler and Type Checker
- [ ] **`crates/fusec/src/sema/check.rs`**: Add `http` builtin namespace and method signatures.
- [ ] **`crates/fusec/src/sema/check.rs`**: Enforce `requires network` capability check for all `http.*` invocations.
- [ ] **`crates/fusec/src/ir/lower.rs`**: Register `http` namespace in builtin name lowering.
- [ ] **`crates/fusec/src/bin/fuse_lsp/completion.rs`**: Add LSP signature help and completions for the new `http` functions.

## Milestone 3: AST Runtime Implementation
- [ ] **`crates/fusec/src/interp/mod.rs`**: Implement AST handlers for `http.request`, `http.get`, and `http.post`.
- [ ] Implement the outbound TCP request helper (parsing host, port, path, headers) reusing the style of the existing proxy path if applicable.

## Milestone 4: Native/JIT Parity
- [ ] **`crates/fusec/src/native/mod.rs`**: Implement `http.*` builtins for the AOT/native non-JIT path.
- [ ] **`crates/fusec/src/native/jit.rs`**: Add JIT hostcalls for the HTTP client.
- [ ] **`crates/fusec/src/native/jit.rs`**: Ensure the HTTP hostcalls are dispatched and whitelisted correctly.

## Milestone 5: Testing & Validation
- [ ] **`crates/fusec/tests/sema_golden.rs`**: Add semantic analysis golden tests covering signatures, capability requirements, and error cases (e.g., missing `requires network`).
- [ ] **`crates/fusec/tests/parity_ast_native.rs`**: Add test cases to guarantee identical behavior between AST, Native, and JIT compilation paths for HTTP requests.
- [ ] **`crates/fusec/tests/html_runtime.rs`**: Add integration tests using a local loopback server to safely test HTTP operations.
- [ ] Validate entire suite using `./scripts/cargo_env.sh cargo test -p fusec`.

## Milestone 6: Documentation & Specification Synchronization (Required)
- [ ] Update `README.md` introducing the HTTP primitives.
- [ ] Update `spec/fls.md` with syntax and signature definitions.
- [ ] Update `spec/runtime.md` emphasizing runtime behavior (blocking, capability traps).
- [ ] Update `governance/scope.md` to formally include basic network I/O in scope.
- [ ] Update `governance/IDENTITY_CHARTER.md` and `governance/EXTENSIBILITY_BOUNDARIES.md` as necessary.
- [ ] Run `./scripts/generate_guide_docs.sh` to compile updated autogenerated guides.
