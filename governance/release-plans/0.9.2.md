# 0.9.2 Patch Plan

Status: Draft
Release line: 0.9.x (patch-safe)

## Objective

Deliver the next LSP/editor quality step: higher signal completions/signatures, better workspace
symbol usability, and additional high-confidence quick fixes.

## Scope guardrails

- Additive editor/tooling improvements only.
- No contract-breaking CLI/runtime changes.
- Preserve existing latency/scalability SLO gates.

## Milestones

### M2.1 Signature help ranking and relevance

Deliverables:

- Improve `textDocument/signatureHelp` target selection and ordering in overloaded/noisy contexts.
- Expand tests for nested call chains and imported symbols.

Acceptance criteria:

- New signature-help tests pass.
- Existing signature-help behavior remains stable for current fixtures.

### M2.2 Workspace symbol search quality

Deliverables:

- Improve workspace symbol matching/ranking beyond plain case-insensitive substring behavior.
- Keep deterministic ordering for equivalent match quality.

Acceptance criteria:

- Workspace-symbol tests cover ranking expectations.
- Search remains within current workspace symbol SLO budget.

### M2.3 Code action: missing capability declarations

Deliverables:

- Add quick fix for sema diagnostics where `requires <capability>` is missing.
- Generate edits at module top-level with deterministic ordering.

Acceptance criteria:

- New code action tests pass for `db`, `network`, `time`, and `crypto` capability cases.
- No false-positive edits on unrelated diagnostics.

### M2.4 Code action: HTML attr migration fixes

Deliverables:

- Add quick fixes for `FUSE_HTML_ATTR_MAP` and `FUSE_HTML_ATTR_COMMA` diagnostics.
- Provide deterministic attr rewrite output.

Acceptance criteria:

- Parser/sema fixture and LSP code-action tests cover both diagnostics.
- Rewrites preserve existing semantics for supported cases.

### M2.5 LSP reliability hardening

Deliverables:

- Strengthen cancellation/burst handling tests around newly added ranking and code actions.
- Keep progressive indexing and cache behavior stable.

Acceptance criteria:

- `lsp_perf_reliability` and `lsp_workspace_incremental` remain green.
- No latency regressions beyond configured thresholds.
