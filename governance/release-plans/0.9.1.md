### M1.4 Completion ranking v2

**Status:** DONE (2026-03-04)

Deliverables:

- Improve completion ordering with context/locality weighting in large dependency graphs.
- Preserve existing stable rank groups as the compatibility baseline.

Acceptance criteria:

- Ranking behavior is covered by tests.
- LSP latency budgets remain within current SLO limits.

#### Summary of what was done

`WorkspaceIndex` (workspace.rs)

- Added `imported_module_uris: HashMap<String, HashSet<String>>` field ŌĆö maps each file URI to the set of file URIs it directly imports (via `import_items` and module aliases)
- Populated in `build_workspace_from_registry` using a pre-built `file_uri_by_idx` map to avoid borrow-checker conflicts
- Serialized as `"imported"` JSON key; deserialized with backward-compatible default

`CompletionCandidate` (completion.rs)

- Added `locality: u8` field (0 = directly imported, 1 = transitively imported; only meaningful for sort_group 2)
- `completion_symbol_sort_group` updated to return `(u8, u8)` and check `directly_imported` set for locality
- `completion_item_json` uses `"02_{locality}_{label}"` format for sort_group 2, giving `"02_0_ŌĆ”"` vs `"02_1_ŌĆ”"`
- External import-binding defs are skipped during candidate collection ŌĆö the original def in the source module carries correct locality (avoids transitive import being falsely promoted by an intermediary's import def)
- Sort and upsert both consider locality as secondary key after sort_group

`lsp_completion_rank.rs`

- Added `lsp_completion_ranking_locality_weights_directly_imported_modules`: dep_util ŌåÉ dep_a ŌåÉ main.fuse chain; asserts `alpha_extra` (dep_a, direct) ŌåÆ `02_0_`, `util_fn` (dep_util, transitive) ŌåÆ `02_1_`

**Check:** All 4 ranking tests pass; `lsp_completion_symbols_and_member_methods` passes; no regressions.
