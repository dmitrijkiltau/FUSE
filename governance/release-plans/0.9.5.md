# 0.9.5 Patch Plan

Status: Draft
Release line: 0.9.x (patch-safe)

## Objective

Ship a polished `0.9.x` capstone patch: diagnostics and example ergonomics improvements, reliability
hardening, and a clean release execution with full verification.

## Scope guardrails

- No breaking language/runtime/tooling behavior changes.
- Keep semantic authority in parser/frontend + sema; no backend-specific semantics.
- Focus on additive UX and release hardening work.

## Milestones

### M5.1 Typed query diagnostics polish

Deliverables:

- Improve compiler diagnostics for typed query mismatch cases (`select([...])`, field mapping, type arg).
- Add machine-readable diagnostic codes where missing.

Acceptance criteria:

- `sema_golden` and typed-query runtime tests cover improved messages/codes.
- No runtime semantics change for query execution paths.

### M5.2 Example modernization (reference-service + db_query_builder)

Deliverables:

- Update examples to emphasize typed-query-first patterns and current best practices.
- Keep docs/examples aligned with current `0.9.x` behavior and release tag defaults.

Acceptance criteria:

- Example checks pass via existing example gates.
- Reference-service docs and compose defaults reflect the `0.9.5` line.

### M5.3 Component/UI ergonomics guidance

Deliverables:

- Add practical guidance for component `attrs`/`children` usage patterns.
- Clarify strict boundary-safe UI composition conventions in guide docs.

Acceptance criteria:

- Guide docs regenerate cleanly.
- Guidance matches current `component` semantics without new language features.

### M5.4 Reliability closure pass

Deliverables:

- Run reliability repeat gate and triage any non-deterministic failures.
- Resolve or time-box flaky issues with explicit ownership and expiry.

Acceptance criteria:

- `./scripts/reliability_repeat.sh --iterations 2` passes on release candidate.
- No indefinite waivers on release-blocking paths.

### M5.5 Final release execution and verification

Deliverables:

- Execute full preflight, artifact packaging, publish flow, and post-publish validation.
- Ensure migration/docs/changelog entries are complete for `0.9.5`.

Acceptance criteria:

- `./scripts/release_preflight.sh 0.9.5` passes.
- Release artifacts publish and verify successfully (checksums + integrity outputs).
- `CHANGELOG.md` contains finalized `0.9.5` section.
