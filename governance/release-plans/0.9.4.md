# 0.9.4 Patch Plan

Status: Implemented, validation in progress
Release line: 0.9.x (patch-safe)

## Objective

Strengthen release artifact integrity and provenance automation in CI while preserving the current
artifact contract and deterministic metadata behavior.

## Scope guardrails

- No artifact naming contract breaks (`fuse-cli-*`, `fuse-aot-*`, `fuse-vscode-*` remain stable).
- Keep checksum and metadata generation behavior backward compatible.
- Add verification layers; do not remove existing release gates.

## Progress snapshot

- Implemented `scripts/generate_release_sboms.sh` to emit deterministic SPDX JSON SBOMs for all
  CLI, AOT, and VSIX payload archives.
- Implemented `scripts/generate_release_provenance.sh` to emit tagged-release provenance with
  commit, ref, tag, workflow path/name/ref, actor, run id, run attempt, checksum manifest, and
  SBOM linkage.
- Implemented `scripts/sign_release_manifest.sh` and wired `.github/workflows/release-artifacts.yml`
  to sign `SHA256SUMS` on aggregate runs and `release-provenance.json` on tagged runs via GitHub
  OIDC keyless `cosign`, using deterministic sidecar names.
- Implemented `scripts/verify_release_integrity.sh` and replaced ad hoc workflow checks so dry-run
  and post-publish verification now fail on checksum, SBOM, provenance, metadata, signature, or
  payload-layout drift.
- Added `scripts/release_integrity_regression.sh` and wired it into `scripts/release_smoke.sh` to
  keep the new integrity bundle logic under local regression coverage.
- Updated `README.md`, `ops/RELEASE.md`, `ops/AOT_RELEASE_CONTRACT.md`, and
  `scripts/generate_release_notes.sh` to document new release assets and downloader verification
  commands.
